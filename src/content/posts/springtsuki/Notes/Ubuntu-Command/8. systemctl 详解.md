# systemctl 指令详解

systemctl 是 Linux 系统上用于管理系统服务（systemd单元）的命令行工具。它可以用来**启动、停止、重启、禁用、启用、查询和管理系统服务的状态**。
以下是一些常用的systemctl命令：

```
systemctl start servicename：启动一个服务。
systemctl stop servicename：停止一个服务。
systemctl restart servicename：重启一个服务。
systemctl enable servicename：设置一个服务为开机自启动。
systemctl disable servicename：禁用一个服务的开机自启动。
systemctl status servicename：查看一个服务的状态。
systemctl list-units --type=service：列出当前系统上所有的服务单元。（此条很重要哦）
```

因为如何查看当前系统上所有的服务单元的命令很重要，所以单独列出！

**systemctl list-units --type=service**

如果是当前启用状态下的，需要加上status=enable的参数！

**systemctl list-units --type=service --state=running**



systemd是Linux系统中的一个初始化系统和系统管理器，它负责启动系统中的各个进程和服务。systemd使用单元文件（unit files）来描述系统中的各种资源，包括服务、挂载点、设备等。当使用systemctl启动一个服务时，systemd会根据该服务的单元文件来启动相应的进程，并托管该进程的生命周期。

而在启动一个服务时，systemd会按照服务单元文件中定义的配置启动相应的进程，并根据配置的依赖关系来确保服务的正确启动顺序。这样可以提高系统的并行启动能力，提高系统启动的效率和稳定性。

### 2. 服务单元文件的结构

这里使用了GPT生成了一个文件，并逐条简介它的文件结构！

```
[Unit]
Description=Your service description
After=network.target

[Service]
ExecStart=/path/to/your/executable
ExecStop=/path/to/your/stop-script
WorkingDirectory=/path/to/working/directory
User=youruser
Group=yourgroup
Restart=always
RestartSec=3
Environment="ENV_VAR=value"
StandardOutput=journal
StandardError=journal

[Install]
WantedBy=multi-user.target
```

#### 1. **[Unit] 部分**

该部分用于定义服务的基本描述和依赖关系。

- `Description`：描述服务的简短信息，通常是服务的功能或用途。
- `Documentation`：可以指定服务相关的文档或 URL。
- `After`：指定服务启动时需要等到其他服务或目标已经启动。比如 `network.target` 表示服务需要在网络服务启动后才启动。
- `Before`：指定该服务需要在其他服务启动之前启动。
- `Requires`：指定服务依赖的其他服务，确保它们在服务启动前运行。
- `Wants`：指定其他服务，和 `Requires` 类似，但不强制要求这些服务必须启动。

#### 2. **[Service] 部分**

该部分用于定义服务如何启动、停止、重启等行为。

- `ExecStart`：定义服务的启动命令或可执行文件的路径。
- `ExecStop`：定义服务停止时执行的命令。
- `ExecReload`：定义服务重载时执行的命令（可选）。
- `WorkingDirectory`：指定服务的工作目录。
- `User`：指定运行该服务的用户。
- `Group`：指定运行该服务的用户组。
- `Restart`：定义服务的重启策略。常见的选项包括：
  - `no`：不自动重启。
  - `always`：服务停止后总是重启。
  - `on-failure`：当服务异常退出时重启。
  - `on-abort`：当服务由于信号异常退出时重启。
- `RestartSec`：定义重启服务的延迟时间（单位：秒）。
- `Environment`：为服务指定环境变量。可以使用多个 `Environment` 行定义多个环境变量。
- `StandardOutput`：指定标准输出的处理方式（如 `journal` 表示输出到 `journal`，也可以指定文件路径）。
- `StandardError`：指定标准错误的处理方式。
- `User` 和 `Group`：指定以哪个用户和用户组身份运行服务。

#### 3. **[Install] 部分**

该部分用于定义服务的安装和启用策略。

- `WantedBy`：定义服务启动时需要的目标（target）。`multi-user.target` 是常见的目标之一，表示在多用户模式下启动服务。
- `RequiredBy`：指定服务是某个目标的依赖项，通常用于定义服务的安装顺序。
- `Alias`：为服务指定别名。
