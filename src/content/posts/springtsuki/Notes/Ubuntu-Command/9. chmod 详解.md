# chmod详解

那么这里首先引入一下权限与参数含义的概念吧，要知道的是文件权限可以用字母和数字两种形式表示。



### 1. 字母表示权限

```
rwx
```

r - 代表读取

w - 代表写入

x - 代表执行



所以可以使用 ls -l 指令查询此文件所拥有的权限

```
ls -l /home/qianyue/linux/changemod.txt
```

可以得到查询的结果为

```
-rw-r--r-- 1 root root 0  3月 24 10:52 /home/qianyue/linux/changemod.txt
```

详细解读：

- 第一个字符 `-` 表示这是一个普通文件（如果是目录，则是 `d`）。
- `rw-`：所有者（`root`）具有读（`r`）和写（`w`）权限，没有执行权限（`-`）。
- `r--`：同组用户（`root`）具有读权限（`r`），没有写权限和执行权限（`--`）。
- `r--`：其他用户具有读权限（`r`），没有写权限和执行权限（`--`）。



如果使用 chmod 修改了权限，例如

```
chmod 777 /home/qianyue/linux/changemod.txt # 这里是给予文件权限
chmod -R 777 /home # -R 参数给予文件夹权限
```

然后再看一下

```
ls -l /home/qianyue/linux/changemod.txt # -l 参数查看文件权限
ls -ld /home # -ld 参数查看文件夹权限
# 查询结果为
-rwxrwxrwx 1 root root 0  3月 24 10:52 /home/qianyue/linux/changemod.txt
```

可以查看到所有用户都拥有了读写与执行的权限

### 2. 数字表示权限

数字方式通过使用三位数来表示文件的权限。每个数字表示一个类别的权限：所有者、用户组和其他用户的权限。每个权限是根据其对应的数字值进行设置的：

- `r` = 4（读取权限）
- `w` = 2（写入权限）
- `x` = 1（执行权限）
- `-` = 0（没有权限）

而每组用户所拥有的权限则为这个数字的结果。

例如 所有者 拥有 读与写 没有执行的权限，所以权限标识符为：4+2 = 6

第二例，例如仅拥有 写与执行 的权限，权限标识符为： 2+1 = 3 

所以所有的权限数字标识符分别为：0、1、3、5、6、7



所以继续看前方的 chmod 777

因为是 三个数字，三个数字 则分别表示了 所有者、同组用户、其他用户 所应有的权限



### 3. 用户参数

在 chmod 命令中，可以使用权限修改符号对用户组添加或移除权限。

#### 权限修改符号：

- **`+`**：添加权限。
- **`-`**：去除权限。
- **`=`**：设置精确的权限，移除其他权限。



当然，也需要一个可以分别表示用户组的字母标识符



#### 常见的用户参数包括：

#### 1. **`u`（user）**

- 代表 **文件的所有者**，即文件的用户。
- 示例：`u+x` 表示给文件的所有者添加执行权限。

#### 2. **`g`（group）**

- 代表 **文件所属的用户组**。
- 示例：`g-w` 表示去除文件所属用户组的写权限。

#### 3. **`o`（others）**

- 代表 **文件的其他用户**，即既不是文件所有者也不是所属用户组的用户。
- 示例：`o=r` 表示将文件的其他用户权限设置为只读。

#### 4. **`a`（all）**

- 代表 **所有用户**，即文件的所有者、用户组和其他用户。
- 示例：`a+x` 表示为所有用户添加执行权限。
- `a` 是 `u`、`g` 和 `o` 的组合，作用于所有用户。



举个例子

```
chmod u+r changemod.txt
```

此含义就是给此文件的所有者增加 读取 的权限
