# free与top指令详解

### 1. free

```
free
```

free 是一个快照机制，输入指令后会读取瞬间的资源使用情况。

输出结果如下：

```
               total        used        free      shared  buff/cache   available
内存：       8082212     1468820     5482464       51724     1447952     6613392
交换：       3958780           0     3958780
```



这个可能不太好阅读，因为没有相关的衡量单位，所以可以使用 -h 参数即可。

```
root@qianyue-VMware-Virtual-Platform:/home# free -h
               total        used        free      shared  buff/cache   available
内存：         7.7Gi       1.4Gi       5.2Gi        50Mi       1.4Gi       6.3Gi
交换：         3.8Gi          0B       3.8Gi
```

这样就更好阅读了



### 2. top

```
top
```

`top` 命令用于实时显示系统进程和资源使用情况。它提供了关于 CPU、内存和交换空间使用情况的动态视图。

简言之就是一个实时的监控器。



#### 常见快捷键（注意大小写区分）：

- **q**：退出 `top`。
- **P**：按 CPU 使用率排序进程。
- **M**：按内存使用量排序进程。
- **T**：按运行时间排序进程。
- **k**：终止进程，按进程 ID 输入。
- **r**：更改进程优先级（即调整进程的 "nice" 值）。
- **1**：显示每个 CPU 的使用情况（如果是多核 CPU）。



#### 带 `-u` 参数：

```
top -u <username>
```

此命令显示特定用户的进程。

#### 带 `-p` 参数：

```
top -p pid
```

此命令显示特定进程 ID（PID）的进程信息。

#### 带 `-d` 参数：

```
top -d 2
```

此命令将更新周期设定为每 2 秒更新一次，适合动态监控。
