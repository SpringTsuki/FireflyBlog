# 2. 盘灵古域 - 梦回盘灵：刷怪机制研究

## 1. 刷怪原理

使用刷怪箱生成不会下落的**屏障**实体方块，并使用tick检测，执行生成怪物的函数，并移除其自身。

并在生成僵尸时，使用 loot_table 控制怪物掉落的战利品。



## 2. 代码实现

```mcfunction
entity_tick.mcfunction 下：

execute as @s[type=falling_block] run function pld:monsters/pos

pos.mcfunction 下：

execute as @s[tag=zombie] at @s run function pld:monsters/east/zombie

zombie.mcfunction 下：
execute store result score @s monster_count run execute if entity @e[tag=panling,tag=monster,distance=..20]

此行用于限制范围内怪物的数量

execute if score @s monster_count matches ..9 run summon minecraft:zombie ~ ~ ~ {Team:"monster",Tags:["panling","undead","monster","east_zombie"],HandDropChances:[0f,0f],ArmorDropChances:[0f,0f,0f,0f],CustomNameVisible:1b,CustomName:'{"translate":"pl.monster.east.zombie"}',Health:18.0f,DeathLootTable:"pld:east/entity/zombie",Attributes:[{Name:"generic.max_health",Base:18d},{Name:"generic.follow_range",Base:40d},{Name:"generic.movement_speed",Base:0.2d},{Name:"generic.attack_damage",Base:2d}],CanPickUpLoot:false,ArmorItems:[{},{},{},{id:"minecraft:leather_helmet",Count:1b,tag:{Unbreakable:1b}}]}
kill @s
```

## 3. 优点

每一只生成的怪物都可以自定义属性，且通过给予指定的 tag ，为每一类型生成的怪物自定义不同的属性和掉落物品！
