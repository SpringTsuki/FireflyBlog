# dockerfile 详解

这里就用我以前的毕业设计来封装成一个容器作为目标吧！

那么首先需要安装 Python 还有一系列的依赖包，同时配置一个 MySQL。

也就意味着，我们需要搬运两个镜像箱子，一个是负责 Python | Django 的镜像箱，一个是 MySQL的镜像箱。

而后使得两个镜像箱在一个容器里协作运行！



#### 这里就以创建 Mysql 的镜像箱为例子

首先需要新建一个 Dockerfile 文件！

```dockerfile
# 使用 MySQL 镜像
FROM mysql:latest

# 设置环境变量
ENV MYSQL_ROOT_PASSWORD=132120
ENV MYSQL_DATABASE=database
ENV MYSQL_USER=qianyue
ENV MYSQL_PASSWORD=132120

# 暴露端口
EXPOSE 3306
```

那么开始构建镜像吧！

```bash
docker build --t mysql .
```

安装完毕就试着启动一下吧！

```bash
docker run mysql
```

似乎启动后就在后台静默运行了，试着连接一下数据库吧。那么先看一下容器的启动状态吧！

```bash
root@qianyue-VMware-Virtual-Platform:/home/qianyue/develop/mysql# docker ps

CONTAINER ID   IMAGE     COMMAND                   CREATED         STATUS         PORTS                 NAMES
1a1d0d3ac948   mysql     "docker-entrypoint.s…"   3 minutes ago   Up 3 minutes   3306/tcp, 33060/tcp   festive_diffie
```

可以看到数据库的名字叫 festive_diffie ，那么就让我们开始连接吧！

```bash
docker exec -it festive_diffie mysql -u root -p
```

可以将以上的指令分为两块：

1. docker exec -it festive_diffie 此处是意味着连接 festive_diffie  容器。
2. 而后在此容器内执行 mysql -u root -p ，意味着使用mysql连接，并同时使用 -u root 用户登录，并包含 -p 的密码选项

```bash
root@qianyue-VMware-Virtual-Platform:/home/qianyue/develop/mysql# docker exec -it festive_diffie mysql -u qianyue -p
```
