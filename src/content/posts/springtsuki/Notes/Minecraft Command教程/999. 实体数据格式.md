### 1. id  - 字符串

*官方定义：（命名空间ID）实体类型。此数据仅在持久化保存（存储入区块文件、结构模板、蜂巢、蜂箱数据或作为乘客实体保存）时存在。/data无法获取此数据，也无法修改。*

```
# 写法如下
id:<string>

# Eg:
id:"pig"
```

id 代表着实体的类型，说白了就是定义一个 **实体** 是 **什么**，实际上 id 似乎也无法通过指令去定义，因为你不可能使用 summon 生成一个 minecraft:pig ，但是却使用指令定义 它 **不是** minecraft:pig 叭~

所以使用 /data get 自然无法获取，F3 + I 也无法获取实体的 ID（实际上就是 minecraft:pig）

~~但是物品格式的 id 是可以使用指令获取的哦，需要区分开~~

```
summon minecraft:pig ~ ~1 ~ {id:"123"}
```

### 2. Air - 短整型

*官方定义：（-20≤值≤实体最大空气值）当前实体所剩的空气值。在指定环境中每刻增加4直至实体最大空气值；在可窒息环境中逐渐减少直至-20，达到-20后如果实体仍然在可窒息环境中则会受到伤害，并将此值重置为0，继续减少进行循环。此值不存在时游戏默认为实体最大空气值。*

```
# 写法如下
Air:<short>}

# Eg:
Air:-20
```

定义实体当前所剩下的空气值，低于-20会受到窒息伤害，在指定环境中每刻增加4直至实体最大空气值，~~对我来说，这个参数没什么用~~

### 3.  **CustomName** - 字符串  or  NBT复合标签：JSON对象 or NBT数组：JSON数组

*官方定义：（文本组件）当前实体的自定义名称。会出现在玩家的死亡消息与村民的交易界面，以及玩家的光标指向的实体的上方。*

**文本组件** 在另一篇详细的教学文档中，这里不详细说明啦。

```
# 写法如下

# 字符串型
CustomName:<string>
# Eg: 生成一只名字为 pig 的猪
summon minecraft:pig ~ ~1 ~ {CustomName:"pig"}

# JSON 对象
CustomName:{<JSON>}
# Eg: 生成一只名字为 pig 的猪
# 列表的形式并不是这么好用，如果需要高度自定义还是使用NBT标签吧~
summon minecraft:pig ~ ~1 ~ {CustomName:["pig"]}

# JSON 数组
CustomName:[<JSON>]
# Eg: 生成一只名字为 pig 、且字体颜色为红色的、且名字一直显示的猪
summon minecraft:pig ~ ~1 ~ {CustomName:{text:"pig",color:red}}
```

#### 4. **CustomNameVisible** - 布尔型

*官方定义：表示实体是否一直渲染名称。如果为true，那么名称会一直在它们的上方渲染，而不受光标指向的影响。如果实体没有自定义名称，则渲染默认名称。此项不存在时游戏默认为`false`。*

```
# 写法如下
CustomNameVisible:<boolean>
```

让实体一直渲染名称！

```
# Eg: 生成一只名字为 pig 、且字体颜色为红色的、且名字一直显示的猪
summon minecraft:pig ~ ~1 ~ {CustomName:{text:"pig",color:red},CustomNameVisible:true}
```
