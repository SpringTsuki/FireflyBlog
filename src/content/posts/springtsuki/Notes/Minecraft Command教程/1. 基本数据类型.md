在学习基本的指令前，首先需要认识指令中所隐含的基本数据类型。即命名标签中所需要用到的参数（**NBT（Named Binary Tag，又称“二进制命名标签”）**、**SNBT（Stringified Named Binary Tag）**是一种用文本表示的树状数据结构）。

## 1. 基本数据类型

NBT的基本结构是带名称的**标签**。标签可以大致分为数值标签和结构标签；前者表示单个值，后者则容纳其他标签，通过嵌套构建树状数据结构。而SNBT游戏内并不会直接存储，而是需要被转换为程序对象，但这一步并非直接完成，而是以NBT作为中介。

## 2. 常用的数据类型

### 1. 布尔型

`true`表示真，`false`表示假。是指令中非常常用的布尔型标签。这些标签位于嵌套结构的最深处，不能再包含其他标签。

例如：**文本组件** 中的 bold （加粗）标签，则需要用布尔型变量：

```mcfunction
# bold:true 时，则会使文本 加粗
tellraw @s {text:"String", bold:true}
```

![2-1](.\image\2-1.png)

### 2. 整型

非常常见，在定义一些需要数值型属性的时候则会用到。

例如：在 **物品堆叠组件** 中定义 魔咒 的等级时则会用到：

```mcfunction
# efficiency:5 时，即为物品赋予 效率5 的魔咒
give @s minecraft:diamond[enchantments={efficiency:5}]
```

![2-2](.\image\2-2.png)

### 3. 浮点数

非常常见，在定义一些需要数值型属性的时候则会用到。

例如：在使用 tp 指令时，若需要精确定义传送的位置时，则可以：

```mcfunction
# 将玩家传送至 0 50 0.055 的位置
tp @s 0 50 0.055
```

![2-3](.\image\2-3.png)

### 4. 字节数组

非常常见，其实在上方所介绍的 **文本组件** 中的 bold （加粗）标签，一整个 "标签" 其实就是一个 "字节数组"

```mcfunction
# "bold:true" 其实就是一个字节数组
tellraw @s {text:"String", bold:true}
```

### 5. 列表

非常常见，例如上方所介绍的 **物品堆叠组件** 中定义 魔咒 的等级是，在 give 物品后的 **" [  ] "**（方括号）就是一个列表。~~（是不是很像 Python 的列表？）~~

```mcfunction
# efficiency:5 时，即为物品赋予 效率5 的魔咒
give @s minecraft:diamond[enchantments={efficiency:5}]
```

不过，这里的列表较为广义，在 give 指令中，它准确的名字应当为 **数据组件列表（minecraft:item_stack）** ，这里与 **物品堆叠组件** 还是有一些不同的，因为 **物品堆叠组件 **的字段是存储在 **components** 里的，但 **components** 实际上是一个 **复合标签** ，并不是一个 **列表**。仅是在 give 指令中需要以 **数据组件列表** 的形式存在。

### 6. 复合标签

最重量级的来啦，在指令学习中会大量且经常碰到复合标签的应用。**复合标签** 以 **花括号形式 " { } "** 存在。在各类指令中都是需要反复使用的。

例如上文介绍的文本组件，它就是一个 **复合标签**，内部的子标签则定义了各种各样不同的属性。

```mcfunction
# bold:true 时，则会使文本 加粗
# {text:"String", bold:true} 就是一个复合标签
tellraw @s {text:"String", bold:true}
```
