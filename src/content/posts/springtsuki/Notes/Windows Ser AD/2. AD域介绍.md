# AD域控



**Active Directory 域控制器（AD 域控）** 是 Microsoft 提供的一项目录服务，用于集中管理和组织计算机、用户、组以及其他资源。它用于为 Windows 网络环境提供身份验证和授权服务，确保网络资源安全、组织清晰且易于管理。



#### 1. 什么是 Active Directory 域控（AD 域控制器）？

AD 域控制器是 Active Directory 域的核心组件，它负责存储和管理域中的所有对象（如用户、计算机、组、组织单位等），并提供身份验证和授权服务。每个 AD 域控制器包含一个目录数据库，保存有关域中所有资源的信息。

- **域**：是一个逻辑上的分组，用来管理和共享资源（如文件、打印机等）。域内的计算机和用户共享统一的配置和身份验证方式。
- **域控制器（DC）**：是存储并管理 Active Directory 数据库的服务器。域控制器负责接收、验证用户登录请求，并控制访问网络资源。

##### 主要功能：

1. **身份验证与授权**：确保用户和设备的身份，并控制访问权限。
2. **集中式管理**：统一管理网络中的所有用户、计算机和组等。
3. **组策略**：可以通过组策略来配置和强制执行计算机和用户的安全策略。
4. **目录服务**：提供一个层次化的数据库来存储网络对象及其属性。

#### 2. 配置 AD 域控

##### 步骤 1：安装 Active Directory 域服务（AD DS）

1. ##### **准备环境**：

   - 确保你的服务器是 Windows Server 版本（如 Windows Server 2019、2022 等）。
   - 确保服务器有静态 IP 地址，避免因为动态 IP 地址变化导致域控制器连接问题。

2. **添加角色：**

   - 打开 **服务器管理器**，在“管理”菜单中选择“添加角色和功能”。
   - 在“安装类型”选择 **基于角色或基于功能的安装**。
   - 选择 **Active Directory 域服务**（AD DS）角色，然后点击下一步。
   - 按照提示安装 AD DS 角色。安装完成后，系统会提示你将服务器提升为域控制器。

`在这里我偷偷偷了个懒、同时也安装了DNS服务器，以便于后续的配置`



##### 步骤 2：提升服务器为域控制器

1. 安装完成后，返回 **服务器管理器**，在 **通知区域** 中，你会看到提示“配置此服务器为域控制器”的通知，点击它。
2. 选择 **添加新森林**（如果这是你设置的第一个域）或 **添加域控制器到现有域**（如果你已经有其他域控制器）。
3. 配置 **域名**：输入你的域名（例如 `example.com`）。
4. 设置 **域控制器功能**：
   - 选择你希望的功能级别（可以选择默认的功能级别，也可以根据需求进行调整）。通常建议选择最新版本的功能级别。
   - 设定 **NetBIOS 名称**（这个是传统的域名，通常是你的域名的前缀）。
5. 设置 **恢复模式密码**：为域控制器设置一个安全的 **Directory Services Restore Mode** (DSRM) 密码。这个密码用于在恢复操作中进行域控制器的恢复。
6. **DNS 选项**：如果这是一个新的域，系统会自动为域控制器安装 DNS 服务器。一般情况下，保持默认即可。
7. **路径设置**：指定 AD 数据库、日志文件和 SYSVOL 文件夹的路径，默认路径通常已经足够。
8. 检查配置并开始安装，安装完成后，系统会自动重启。



那么到这里，域控就安装完毕了。
