# 用户与计算机的加域操作

那么这里我们先创建一个全局用于加域的账户吧！



#### 1. 创建全局加域账户

那么首先我们这里先创建一个组织架构，命名为 operator，而后再次组织下创建账户 `qy-admin` ，设定密码后待用。

为了便于管理 **安全组** ，我们就在组织架构 User 下创建一个新的 **安全组**，作用于 **全局域** 。就命名为 `qynet-admin`

而后为了使得此 **安全组** 拥有足够的权限，所以需要它隶属于 **管理员** 的权限身份下。

管理员权限身份一共有两个，一个是名为 **Administrator** 的本地管理员账号，还有一个是 **Domain Admins** 的域管理员账号。

拥有权限后，我们就能使用此账号进行加域或退域的相关操作了。



#### 2. 创建组织架构（OU）图

那么相关的账户创建完毕了，接下来就可以进行加域操作了。

那么首先，我们则可以继续创建一个实打实的组织架构图，例如我们创建一个服务器管理的组织。

而后在此根目录下，创建两个分支组织架构，一个是管理计算机，一个是用于管理用户的账号。



#### 3. 创建计算机与新用户

为了让其他的计算机能够加入此域，我们需要添加一个新的 **计算机** ，也就是添加一个 **计算机名**。

添加完后，我们就可以创建一个新的 **用户** 了，创建完用户后，我们就可以进行加域操作了！



#### 4. 加域

由于是在本地创建的域控服务器，并同时创建了一个 DNS 委派，并将此域名绑定为了域控服务器的域名。

由于本机的IP在内网环境下为 192.168.31.210 ，所以 DNS 服务器的内网IP就是这个。

为了让 DNS服务器 在其他的同一内网环境下的计算机生效，所以我们需要将待加域的网络设置中的 DNS服务器，修改为对应的内网IP。

这样的话，就可以解析对应的域名了。



最后，我们在对应的计算机上进行加域，并使用先前预先创建好的加域用户，即可完成加域。

而后，我们就可以使用创建好的 QIANYUE 用户进行登陆了！
