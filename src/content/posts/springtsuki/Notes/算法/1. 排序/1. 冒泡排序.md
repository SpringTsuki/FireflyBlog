# 冒泡排序

```
arr = [21, 3, 5, 7, 9]
```

这是一段长度为 5 的列表、需要使用冒泡排序法则进行排序。

所以对应的数组下标分别为

```
a[0] = 21
a[1] = 3
a[2] = 5
a[3] = 7
a[4] = 9
```

故得到 **n 的最大值** 即 **列表的长度（4）**。

由于最坏的情况 -> 即最小值在末尾，所以冒泡的最坏情况是需要交换 **4** 次才可以到首位。

根据数学归纳法，也就是需要 **n-1** 次。



所以我们使用 **j** 来表示 **for循环** 内列表的下标，由于 **j+1** 最大为 **4**。所以 **j** 的范围为 **从0至3** 。

根据数学归纳法，也就是 **j 在for循环的最大值，为 n-1 **。

而后，我们以 **j** 的 for循环 内，比较 **arr[j] 与 arr[j+1]** 的大小即可



#### 以下是Python实现的方法：

```python
# 1.1 冒泡排序
# 代码攥写时间: 2025.3.19

test_list = [36, 27, 18, 9, 25, 77, 32, 21, 23, 20]
print(test_list)


def sort(arr):
    # 首先需要获取列表的长度
    arr_long = len(arr)
    # 遍历一次列表
    for n in range(arr_long):
        # 根据最坏情况，需要执行 列表长度减 1 次
        for j in range(arr_long - 1):
            if arr[j] > arr[j + 1]:
                arr[j], arr[j + 1] = arr[j + 1], arr[j]
    return arr


sort(test_list)
print(test_list)
```

#### 运行结果：

```
[36, 27, 18, 9, 25, 77, 32, 21, 23, 20]
[9, 18, 20, 21, 23, 25, 27, 32, 36, 77]
```



那么进一步的，我们可以发现：

#### 在每一次冒泡遍历后，最大的值都会直接排序到队伍的末尾。

初始列表和第一次遍历后的结果如下：

```
[36, 27, 18, 9, 25, 77, 32, 21, 23, 20]
[27, 18, 9, 25, 36, 32, 21, 23, 20, 77]
```

所以我们不难发现，**在每一次遍历后，都无需再次遍历最后一个元素** ，所以，在接下来的遍历中，都可以少遍历一个元素。

因为 **n** 是控制内部 for循环 遍历的 列表下标，所以我们只需要减去一次 **n** ，就可以在每一次循环后不遍历末尾的元素了。

```
在第一次外层循环时，i = 0，内层循环将会遍历 arr[0] 到 arr[9]，比较并将最大的元素放到最后的位置。完成后，末尾元素已经排序完成。
在第二次外层循环时，i = 1，内层循环只需要遍历 arr[0] 到 arr[8]，因为 arr[9] 位置已经是有序的。
以此类推，直到整个数组有序。
通过这种方式，冒泡排序的比较次数逐步减少，提高了效率。
```

故代码可优化为：

```python
def sort(arr):
    # 首先需要获取列表的长度
    arr_long = len(arr)
    # 遍历一次列表
    for n in range(arr_long):
        # 根据最坏情况，需要执行 列表长度减 1 次
        for j in range(arr_long - n - 1):
            if arr[j] > arr[j + 1]:
                arr[j], arr[j + 1] = arr[j + 1], arr[j]
    return arr
```

#### 运行结果：

```
[36, 27, 18, 9, 25, 77, 32, 21, 23, 20]
[9, 18, 20, 21, 23, 25, 27, 32, 36, 77]
```



#### 当然，我们还可以继续优化

如果列表没有发生任何一次冒泡交换，是不是意味着列表已经排序完成了呢？

所以我们可以再加入一个flag，检测一次遍历中是否没有发生交换，如果没有发生交换则意味着已经排序完成。用break直接跳出循环即可。

故可优化为：

```python
def sort(arr):
    # 首先需要获取列表的长度
    arr_long = len(arr)
    for n in range(arr_long):
        result_flag = False
        for j in range(arr_long - n - 1):
            if arr[j] > arr[j + 1]:
                arr[j], arr[j + 1] = arr[j + 1], arr[j]
                result_flag = True
        if not result_flag:
            break
    return arr
```



#### 时间复杂度

- **最坏情况**：O(n²)，当列表是逆序时。
- **最好情况**：O(n)，当列表已经有序时。
- **平均情况**：O(n²)。

#### 空间复杂度

- O(1)，因为冒泡排序是原地排序算法，不需要额外的存储空间。
