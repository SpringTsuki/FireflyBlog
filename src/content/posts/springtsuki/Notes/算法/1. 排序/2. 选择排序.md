# 选择排序

其实从这里开始我也不好具体解释这个排序的方法。

简单来说就是一个列表，有很多的元素。

需要从头遍历至尾部，在此其中找到最小的值，记录下标后，根据下标找到这个元素，和第一个进行交换。

进而以此类推。



#### 以下为Python实现的方法：

```python
# 1.2 选择排序
# 代码攥写时间: 2025.3.19

test_list = [36, 27, 18, 9, 25, 77, 32, 21, 23, 20]
print(test_list)


def sort(arr):
    arr_long = len(arr)
    for n in range(arr_long):
        # 这里我们起初假定最小的就是第n个（起初为第一个），而后再让它依次作比较
        minIndex = n
        for j in range(n, arr_long):
            # 如果队列里的元素比这个假定最小值的元素还要小，那么就让这个下标更新为这个元素的列表下标
            if arr[j] < arr[minIndex]:
                minIndex = j
        # 遍历一次完成后，根据最小的下标找到此元素，让它与队列里的第n个元素进行交换（首次为第一个，进而以此类推）
        arr[n], arr[minIndex] = arr[minIndex], arr[n]
        print(arr)
    return arr


sort(test_list)
print(test_list)
```



那么进一步的，我们可以发现优化的点。

#### 我们不难发现，在每一次遍历中，已知最小的下标（minIndex）若与 n 相同，则不需要自身与自身比较。

所以，我们可以在第二层嵌套的for循环（进行比较功能）中，将内部循环的 **n** 从 **n+1** 开始，这样就避免了比较自身

```python
 for j in range(n+1, arr_long):
```

#### 同时也不难发现，如果我们的minIndex已经是最小值了，则不需要进行交换

所以我们可以在交换时添加一个条件，若 minIndex 与 n 值不同时，再进行交换即可。

```python
if minIndex != n:
    arr[n], arr[minIndex] = arr[minIndex], arr[n]
```



所以优化后的代码如下：

```python
# 1.2 选择排序
# 代码攥写时间: 2025.3.19

test_list = [36, 27, 18, 9, 25, 77, 32, 21, 23, 20]
print(test_list)


def sort(arr):
    arr_long = len(arr)
    for n in range(arr_long):
        # 这里我们起初假定最小的就是第n个（起初为第一个），而后再让它依次作比较
        minIndex = n
        for j in range(n+1, arr_long):
            # 如果队列里的元素比这个假定最小值的元素还要小，那么就让这个下标更新为这个元素的列表下标
            if arr[j] < arr[minIndex]:
                minIndex = j
        # 遍历一次完成后，根据最小的下标找到此元素，让它与队列里的第n个元素进行交换（首次为第一个，进而以此类推）
        # 若代表着最小值的下标minIndex已经与自身n相同了，则不需要交换了，直接进入下一轮循环
        if minIndex != n:
            arr[n], arr[minIndex] = arr[minIndex], arr[n]
    return arr


sort(test_list)
print(test_list)
```



#### 时间复杂度

- **最坏情况**：O(n²)，无论输入数据是否有序，都需要进行 n(n-1)/2 次比较。
- **最好情况**：O(n²)，即使列表已经有序，仍需进行相同数量的比较。
- **平均情况**：O(n²)。

------

#### 空间复杂度

- O(1)，选择排序是原地排序算法，不需要额外的存储空间。
